SpringBoot中打日志,让排查不再错误不再难

2019.08.19

使用日志在排查错误的时候可以更快的定位到错误所在的位置,SpringBoot已经帮我们集成了相关的日志模块

参考

  • SpringBoot-Log(官方文档)

版本

  • SpringBoot版本: 2.0.4.RELEASE

背景

在使用SpringBoot开发项目的过程中，日志方面直接使用全部默认的配置的。部署在服务器上直接是

nohup java -jar xxx.jar &

所有的日志都输出到nohup.out中。不符合有追求的程序员的开发方式，所以得改。并且在后续的工作中，使用ELK可以实时的分析日志。

开搞

在官方的文档中，SpringBoot使用的Commons-logging作为默认的日志框架，但是同时也预留了其他的日志框架的实现。我们使用日志主要是为了完成一下的功能：

  • 记录程序执行的细节
  • 给程序出错的时候排查问题的所在
  • 结合其他工具来完成一些功能(e.g:使用ELK实时分析日志)
